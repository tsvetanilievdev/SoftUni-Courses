Целта на app.js - с подготовката на модулите

Целта на отделните модули(catalog, details, create от примера cookbook) да отговарят само за:
Управление на изгледа(view control) - визуализация на съдържанието и закачането на Event Listeners за разни функции 

Отделени модули, автономни, абстрактни(navigation, rendering, API calls), които се занимават -

view control VS rendering
view control - управление на събития, като се извика някоя функция да се извършват някакви операции
rendering - визуализация на екрана

Примери за изолирани модули:
 - Backend API - специфичен спрямо услугата, която се ползва(напр. модул за работа с Firebase, друг модул за работа Backendless) - извършване на ЗАЯВКИ
 - Request logic - специфичната за приложението бизнес логика - пример от cookbook ---> модул който взима всички рецепти, като ще викне метод от модулите за Backend API(използваме модула за Firebase, ако се наложи смяна на сървиса се използма напр. модула за Backendless..)
 - Data manipulation - специфичната за приложението бизнес логика - напр. имаме МАСИВ от обекти, като искаме от тези обекти да извадим само заглавията. 
 - UI display and control - визуализация потребителския интерфейс и негового управление
 - Utility function - всички неща свързани с 'удобства', напр. функция за някакво сравнение(sort)
